## 一、创建飞书应用

### 步骤1：进入飞书开放平台

1. 打开浏览器，访问：**https://open.feishu.cn**
2. 点击右上角 **登录**
3. 使用您的飞书账号扫码登录

![开放平台首页](./images/feishu-1.png)

---

### 步骤2：创建企业自建应用

1. 登录后，点击 **控制台**
2. 点击 **创建企业自建应用**
3. 填写应用信息：

```
应用名称：配送管理系统
应用描述：团队配送管理系统，用于管理客户订单和配送记录
应用图标：上传一个图标（可选）
```

4. 点击 **创建**

![创建应用](./images/feishu-2.png)

---

### 步骤3：进入应用详情

创建成功后，会自动进入应用详情页面
- 如果没有自动进入，点击应用名称进入

---

## 二、获取配置信息

### 步骤1：获取 App ID 和 App Secret

1. 在应用详情页面
2. 左侧菜单点击 **凭证与基础信息**
3. 在右侧可以看到：

```
App ID: cli_xxxxxxxxxxxxxxxxxxxx
App Secret: xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

4. 点击复制按钮，保存这两个值

**⚠️ 重要：App Secret 只显示一次，请立即保存！**

![获取凭证](./images/feishu-3.png)

---

### 步骤2：获取多维表格 Token

#### 2.1 打开多维表格
1. 打开飞书客户端或网页版
2. 进入您的 **配送管理系统多维表格**

#### 2.2 查看地址栏
浏览器地址栏格式：
```
https://xxx.feishu.cn/base/bascnKJDH1234567890abcdef?table=tblXXXXXXXXXXXX
                         ↑ 这就是表格Token
```

#### 2.3 复制表格Token
- 表格Token是以 `bascn` 开头的字符串
- 例如：`bascnKJDH1234567890abcdef`
- 复制保存这个值

![表格Token](./images/feishu-4.png)

---

### 步骤3：获取各个表的ID

#### 3.1 切换到客户表
1. 在多维表格中，点击 **客户表** 标签
2. 查看浏览器地址栏：
```
https://xxx.feishu.cn/base/xxx?table=tblXXXXXXXXXXXX&view=vewXXXXXXXXXXXX
                                      ↑ 这就是表ID
```
3. 表ID是以 `tbl` 开头的字符串
4. 复制保存这个值

#### 3.2 依次获取其他表ID
按照相同方法，获取：
```
✅ 客户表ID：tblXXXXXXXXXXXX
✅ 配送表ID：tblXXXXXXXXXXXX
✅ 假期表ID：tblXXXXXXXXXXXX
✅ 暂停表ID：tblXXXXXXXXXXXX
```

**⚠️ 注意：每个表的ID都不一样！**

## 三、开通权限

### 步骤1：进入权限管理

1. 在飞书开放平台，进入您的应用
2. 左侧菜单点击 **权限管理**

![权限管理](./images/feishu-5.png)

---

### 步骤2：开通多维表格权限

#### 2.1 搜索权限
在搜索框输入：`bitable`

#### 2.2 开通权限
找到以下权限，点击 **申请权限**：

```
✅ bitable:record:read
   名称：获取记录
   说明：查看多维表格记录

✅ bitable:record:write
   名称：更新记录
   说明：编辑多维表格记录

✅ bitable:app
   名称：查看多维表格
   说明：获取多维表格元数据
```

![开通权限](./images/feishu-6.png)

---

### 步骤3：提交审批

1. 选择完权限后，点击 **申请发布**
2. 填写申请理由：
```
用于团队配送管理系统，需要读取和编辑多维表格数据
```
3. 点击 **提交**

**⚠️ 注意：如果您是企业管理员，可以自己审批；否则需要等待管理员审批**

---

## 四、发布应用

### 步骤1：创建版本

1. 左侧菜单点击 **版本管理与发布**
2. 点击 **创建版本**
3. 填写版本信息：

```
版本号：1.0.0
更新说明：首次发布，包含配送管理基础功能
```

4. 点击 **保存并提交审核**

![创建版本](./images/feishu-7.png)

---

### 步骤2：等待审核

- 审核时间：通常几小时到1天
- 可以在 **版本管理与发布** 页面查看审核状态

---

### 步骤3：发布应用

审核通过后：
1. 在版本列表中，找到审核通过的版本
2. 点击 **发布**
3. 确认发布

**✅ 发布成功后，应用就可以使用了！**

## 五、配置信息汇总

### 您现在应该有的配置信息：

```json
{
  "appId": "cli_xxxxxxxxxxxxxxxxxxxx",
  "appSecret": "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
  "bitableToken": "bascnKJDH1234567890abcdef",
  "customerTableId": "tblXXXXXXXXXXXX",
  "deliveryTableId": "tblXXXXXXXXXXXX",
  "holidayTableId": "tblXXXXXXXXXXXX",
  "pauseTableId": "tblXXXXXXXXXXXX"
}
```

### 配置信息说明：

| 配置项 | 说明 | 获取位置 |
|--------|------|----------|
| **App ID** | 应用唯一标识 | 开放平台 → 应用 → 凭证与基础信息 |
| **App Secret** | 应用密钥 | 开放平台 → 应用 → 凭证与基础信息 |
| **表格Token** | 多维表格标识 | 多维表格URL |
| **客户表ID** | 客户表标识 | 多维表格 → 客户表 → URL |
| **配送表ID** | 配送表标识 | 多维表格 → 配送表 → URL |
| **假期表ID** | 假期表标识 | 多维表格 → 假期表 → URL |
| **暂停表ID** | 暂停表标识 | 多维表格 → 暂停表 → URL |

## 七、常见问题

### Q1: App Secret 忘记了怎么办？

**A:** 
1. 进入应用 → 凭证与基础信息
2. 点击 App Secret 旁边的 **重置**
3. 会生成新的 App Secret
4. **⚠️ 重置后旧的Secret立即失效，需要重新配置**

---

### Q2: 权限申请一直待审批？

**A:**
- 联系企业管理员审批
- 或者让管理员给您管理员权限

---

### Q3: 找不到多维表格Token？

**A:**
- 确保您打开的是多维表格（不是普通文档）
- 多维表格URL格式：`https://xxx.feishu.cn/base/xxxxxx`
- Token 在 `/base/` 后面

---

### Q4: 表ID都一样怎么办？

**A:**
- 表ID不可能一样
- 确保切换到了不同的数据表标签
- 每次切换标签后，URL中的 `table=` 后面的值会变化

---

### Q5: 应用发布后还是无法使用？

**A:**
检查以下几点：
1. ✅ 权限是否已开通
2. ✅ 应用是否已发布（不是草稿状态）
3. ✅ 配置信息是否正确
4. ✅ 多维表格是否属于同一企业

---

### Q6: 如何添加更多权限？

**A:**
1. 进入应用 → 权限管理
2. 搜索并申请新权限
3. 创建新版本
4. 发布新版本

---

### Q7: 可以用别人的应用吗？

**A:**
- ✅ 可以，但需要：
  1. 对方应用的 App ID 和 App Secret
  2. 对方应用已开通相应权限
  3. 对方应用已发布
- ⚠️ 不推荐，建议自己创建应用

## 八、安全建议

### 配置信息安全

#### ✅ 推荐做法：
1. **只在团队内部分享配置信息**
2. **定期更换 App Secret**（每3-6个月）
3. **限制应用权限范围**（只开通必要权限）
4. **监控应用使用情况**（开放平台可查看）

#### ❌ 不要做：
1. 不要把配置信息发到公开群组
2. 不要把配置信息提交到公开代码仓库
3. 不要开通不必要的权限
