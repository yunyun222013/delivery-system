## 🚨 常见问题快速检查

### 问题：Railway部署失败
**检查：**
- [ ] requirements.txt 是否包含不可安装的包？
- [ ] Python版本是否正确（3.11.4）？
- [ ] Procfile 格式是否正确？

**解决：**
确保使用简化后的 requirements.txt：
```
fastapi==0.104.1
uvicorn==0.24.0
pydantic==2.5.0
requests==2.31.0
python-multipart==0.0.6
python-dotenv==1.0.0
```

---

### 问题：前端无法连接后端
**检查：**
- [ ] 后端是否部署成功？
- [ ] API地址是否正确？
- [ ] 地址末尾是否有多余的斜杠？

**解决：**
1. 测试后端：访问 `https://后端地址/health`
2. 确认返回 `{"status": "ok"}`
3. 前端配置时不要加末尾斜杠

---

### 问题：登录失败
**检查：**
- [ ] 用户名密码是否正确？
- [ ] 后端是否正常运行？
- [ ] 环境变量是否配置？

**解决：**
1. 确认后端健康检查通过
2. 检查Railway日志查看错误
3. 确认默认账号：admin / admin123

---

### 问题：操作失败
**检查：**
- [ ] 飞书配置是否正确？
- [ ] 飞书应用是否已发布？
- [ ] 权限是否已开通？

**解决：**
1. 检查Railway环境变量
2. 确认飞书应用已发布
3. 确认已开通必要权限
