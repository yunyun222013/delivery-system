## 第一部分：准备工作

### 步骤1.1：准备GitHub账号

#### 1.1.1 检查是否已有账号
- 访问：https://github.com
- 如果能登录，跳到步骤1.2

#### 1.1.2 注册GitHub账号（如需要）
1. 访问：https://github.com/signup
2. 填写信息：
   ```
   Username: 输入用户名（英文）
   Email: 输入邮箱
   Password: 输入密码
   ```
3. 点击 **Create account**
4. 验证邮箱
5. 选择 **Free** 计划
6. 完成

**⏱️ 预计时间：5分钟**

---

### 步骤1.2：准备Railway账号

#### 1.2.1 登录Railway
1. 访问：https://railway.app
2. 点击右上角 **Login**
3. 选择 **Login with GitHub**
4. 授权Railway访问您的GitHub

#### 1.2.2 确认登录成功
- 看到您的用户头像
- 可以点击 **Start a New Project**

**⏱️ 预计时间：1分钟**

---

### 步骤1.3：获取飞书配置信息

#### 1.3.1 创建飞书应用

**步骤：**
1. 访问：https://open.feishu.cn
2. 用飞书扫码登录
3. 点击 **控制台**
4. 点击 **创建企业自建应用**
5. 填写信息：
   ```
   应用名称：配送管理系统
   应用描述：团队配送管理系统
   应用图标：上传一个图标（可选）
   ```
6. 点击 **创建**

**⏱️ 预计时间：2分钟**

#### 1.3.2 获取App ID和App Secret

**步骤：**
1. 进入刚创建的应用
2. 左侧菜单点击 **凭证与基础信息**
3. 找到：
   ```
   App ID: cli_xxxxxxxxxxxx
   App Secret: xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   ```
4. 点击复制按钮，保存这两个值

**⚠️ 重要：App Secret只显示一次，立即保存！**

**⏱️ 预计时间：1分钟**

#### 1.3.3 开通权限

**步骤：**
1. 左侧菜单点击 **权限管理**
2. 在搜索框输入：`bitable`
3. 找到以下权限，点击 **申请权限**：
   ```
   ✅ bitable:record:read - 获取记录
   ✅ bitable:record:write - 更新记录
   ✅ bitable:app - 查看多维表格
   ```
4. 点击 **申请发布**
5. 填写申请理由：
   ```
   用于团队配送管理系统，需要读取和编辑多维表格数据
   ```
6. 点击 **提交**

**⏱️ 预计时间：2分钟**

#### 1.3.4 发布应用

**步骤：**
1. 左侧菜单点击 **版本管理与发布**
2. 点击 **创建版本**
3. 填写：
   ```
   版本号：1.0.0
   更新说明：首次发布
   ```
4. 点击 **保存并提交审核**
5. 等待审核（通常几小时到1天）
6. 审核通过后，点击 **发布**

**⏱️ 预计时间：审核时间不定，可先进行后续步骤**

#### 1.3.5 获取多维表格Token

**步骤：**
1. 打开飞书客户端或网页版
2. 打开您的配送管理多维表格
3. 查看浏览器地址栏：
   ```
   https://xxx.feishu.cn/base/bascnKJDH1234567890abcdef?table=...
                              ↑ 这就是表格Token
   ```
4. 复制 `bascn` 开头的字符串

**示例：**
```
完整地址：https://abc.feishu.cn/base/bascnABCD1234?table=tblXXXX
表格Token：bascnABCD1234
```

**⏱️ 预计时间：1分钟**

#### 1.3.6 获取各个表的ID

**步骤：**
1. 在多维表格中，点击 **客户表** 标签
2. 查看浏览器地址栏：
   ```
   https://xxx.feishu.cn/base/xxx?table=tblXXXXXXXXXXXX&view=...
                                      ↑ 这就是表ID
   ```
3. 复制 `tbl` 开头的字符串
4. 重复以上步骤，获取所有表ID：
   ```
   客户表ID：tblXXXXXXXXXXXX
   配送表ID：tblXXXXXXXXXXXX
   假期表ID：tblXXXXXXXXXXXX
   暂停表ID：tblXXXXXXXXXXXX
   ```

**⚠️ 注意：每个表的ID都不一样！**

**⏱️ 预计时间：2分钟**

#### 1.3.7 汇总配置信息

**创建一个文本文件，保存以下信息：**
```
========== 飞书配置信息 ==========

App ID: cli_xxxxxxxxxxxx
App Secret: xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

表格Token: bascnXXXXXXXXXXXXXXXX

客户表ID: tblXXXXXXXXXXXX
配送表ID: tblXXXXXXXXXXXX
假期表ID: tblXXXXXXXXXXXX
暂停表ID: tblXXXXXXXXXXXX

==================================
```

**✅ 检查点：**
- [ ] 已获取 App ID
- [ ] 已获取 App Secret
- [ ] 已获取表格Token
- [ ] 已获取4个表的ID
- [ ] 已保存到文件

## 第二部分：上传代码到GitHub

### 步骤2.1：解压文件包

#### 2.1.1 解压下载的文件包
```
文件名：delivery_system_github_upload.tar.gz
```

**Windows系统：**
1. 右键点击文件
2. 选择 **解压到当前文件夹**
3. 或用 WinRAR、7-Zip 解压

**Mac系统：**
1. 双击文件自动解压
2. 或终端运行：`tar -xzf delivery_system_github_upload.tar.gz`

**解压后得到文件夹：**
```
github_upload_package/
├── README.md
├── QUICK_START.md
├── backend/
├── frontend/
└── docs/
```

**⏱️ 预计时间：30秒**

---

### 步骤2.2：创建GitHub仓库

#### 2.2.1 登录GitHub
1. 访问：https://github.com
2. 登录您的账号

#### 2.2.2 创建新仓库
1. 点击右上角 **+** 号
2. 选择 **New repository**

#### 2.2.3 填写仓库信息
```
Repository name: delivery-system
Description: 配送管理系统
Visibility: ✅ Public（公开，才能用GitHub Pages免费）
          或 Private（私有，更安全）

⚠️ 注意：
- 选择 Public：完全免费，但代码公开
- 选择 Private：需要GitHub Pro，但代码私密
```

#### 2.2.4 创建仓库
1. 不要勾选任何初始化选项
2. 点击 **Create repository**

**⏱️ 预计时间：1分钟**

---

### 步骤2.3：上传文件到GitHub

#### 方法一：网页上传（推荐，最简单）

**步骤：**
1. 在刚创建的仓库页面
2. 看到 **Quick setup** 界面
3. 点击 **uploading an existing file**

**或者：**
1. 点击 **Add file**
2. 选择 **Upload files**

**上传文件：**
1. 打开解压后的 `github_upload_package` 文件夹
2. 选择文件夹内的 **所有内容**（不是文件夹本身）
3. 拖拽到GitHub上传区域

**确保上传的内容：**
```
✅ README.md
✅ QUICK_START.md
✅ .gitignore
✅ backend/ 文件夹（包含所有文件）
✅ frontend/ 文件夹（包含index.html）
✅ docs/ 文件夹（包含所有文档）
```

**提交：**
1. 在下方 **Commit changes**
2. 填写提交信息：
   ```
   初始化配送管理系统
   ```
3. 点击 **Commit changes**

**⏱️ 预计时间：2分钟**

#### 方法二：Git命令行（适合熟悉Git的用户）

**步骤：**
```bash
# 1. 进入解压后的文件夹
cd github_upload_package

# 2. 初始化Git
git init

# 3. 添加所有文件
git add .

# 4. 提交
git commit -m "初始化配送管理系统"

# 5. 连接到GitHub仓库
git remote add origin https://github.com/你的用户名/delivery-system.git

# 6. 推送到GitHub
git branch -M main
git push -u origin main
```

**⏱️ 预计时间：1分钟**

## 第三部分：部署后端到Railway

### 步骤3.1：创建Railway项目

#### 3.1.1 登录Railway
1. 访问：https://railway.app
2. 点击 **Login with GitHub**
3. 授权访问

#### 3.1.2 创建新项目
1. 点击 **Start a New Project**
2. 选择 **Deploy from GitHub repo**
3. 如果是第一次使用，点击 **Configure GitHub App**
4. 授权Railway访问您的GitHub仓库
5. 选择 **Only select repositories**
6. 勾选 **delivery-system** 仓库
7. 点击 **Save**

#### 3.1.3 选择仓库
1. 返回Railway
2. 点击 **delivery-system** 仓库

**⏱️ 预计时间：2分钟**

---

### 步骤3.2：配置项目设置

#### 3.2.1 设置根目录
1. 在项目创建页面
2. 找到 **Root Directory** 设置
3. 点击 **Settings**
4. 找到 **Root Directory**
5. 输入：`backend`
6. 点击 **Save**

**⚠️ 重要：这一步很关键！必须设置为 backend**

#### 3.2.2 确认配置
- Root Directory: `backend`
- Railway会自动检测到：
  - requirements.txt
  - Procfile
  - runtime.txt

**⏱️ 预计时间：1分钟**

### 步骤6.1：准备分享信息

创建一个文档，包含以下信息：

```markdown
# 配送管理系统使用说明

## 访问地址
https://用户名.github.io/delivery-system/

## 登录账号
- 管理员：admin / admin123
- 配送员：user1 / user123
- 配送员：user2 / user123
- 配送员：user3 / user123

1. 打开访问地址
2. 在页面底部找到 "后端API地址配置"

4. 点击 "保存API地址"
```
